# Отчёт о применении Schema-Guided Reasoning (SGR)

## Резюме

Все промты в проекте были улучшены с применением методологии **Schema-Guided Reasoning (SGR)** для устранения ошибок из отчёта по тестированию.

## Решённые проблемы из отчёта

| № | Проблема | Решение через SGR |
|---|----------|-------------------|
| 6 | Небольшие речевые ошибки | Добавлен явный checkpoint проверки стиля во всех SGR-схемах |
| 7 | Некорректное отображение (Shift+Enter) | Этап 1 нормализации явно выявляет проблемы форматирования |
| 8 | Дублирование информации | Этап 5 (дедупликация) обязателен для всех агентов |
| 9 | Не сразу начинает генерацию | SGR даёт модели чёткую структуру для немедленного начала |
| 10 | Недетерминированность результатов | 7 обязательных этапов SGR гарантируют одинаковый порядок проверки |

## Преимущества SGR-подхода

### 1. Детерминированность
Одинаковый вход → одинаковый результат благодаря:
- Фиксированным этапам обработки
- Явным правилам верификации
- Этапу дедупликации

### 2. Аудируемость
Каждый шаг проверки явно задокументирован в JSON-схеме:
- Можно восстановить ход рассуждений
- Можно выявить, на каком этапе произошла ошибка
- Можно тестировать отдельные этапы

### 3. Тестируемость
Промежуточные результаты можно валидировать:
- Каждый этап имеет чёткий формат вывода
- Можно создать тестовые наборы для каждого этапа
- Можно измерять точность на каждом этапе

### 4. Консистентность
Все агенты используют единый подход:
- Одинаковая структура этапов
- Единые принципы верификации
- Согласованные правила маршрутизации

## SGR-паттерны в проекте

### Cascade (Каскад)
**Применение:** Последовательное прохождение обязательных этапов
**Файлы:** review-agent, chat-agent, document-agent, orchestrator
**Эффект:** Гарантия полноты проверки

### Cycle (Цикл)
**Применение:** Итеративная проверка каждого элемента по единому чек-листу
**Файлы:** review-agent, document-agent
**Эффект:** Консистентность проверки

### Routing (Маршрутизация)
**Применение:** Классификация перед обработкой
**Файлы:** classifier, orchestrator, chat-agent
**Эффект:** Правильный выбор стратегии обработки

### Verification (Верификация)
**Применение:** Проверка каждого вывода перед финальным ответом
**Файлы:** Все агенты
**Эффект:** Отсев ложных срабатываний

### Deduplication (Дедупликация)
**Применение:** Устранение повторяющихся ошибок
**Файлы:** review-agent
**Эффект:** Отсутствие дублирования в ответе

## Рекомендации по использованию

### Для разработчиков
1. При добавлении нового агента используйте SGR-схему из `types.ts`
2. Каждый новый промпт должен иметь минимум 4 этапа (анализ, обработка, верификация, вывод)
3. Логируйте каждый этап для отладки

### Для тестирования
1. Создайте тестовые наборы для каждого SGR-этапа
2. Тестируйте детерминированность (одинаковый вход → одинаковый выход)
3. Проверяйте полноту прохождения всех этапов

### Для отладки
1. Включите логирование SGR-этапов в консоли
2. Сравнивайте промежуточные результаты этапов
3. Выявляйте, на каком этапе происходит ошибка
